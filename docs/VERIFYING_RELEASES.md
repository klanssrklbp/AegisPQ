# Verifying AegisPQ releases

Every AegisPQ release published by the [`release.yml`] workflow ships with
three layers of authenticity evidence:

1. **`SHA256SUMS`** — a single manifest listing the SHA-256 digest of every
   archive in the release.
2. **Cosign keyless signatures** — each archive and the `SHA256SUMS` manifest
   have a detached signature (`.sig`) and an ephemeral X.509 signing
   certificate (`.pem`) issued by [Sigstore](https://www.sigstore.dev/) via
   GitHub Actions' OIDC identity. The signing event is recorded in the public
   [Rekor transparency log](https://docs.sigstore.dev/logging/overview/).
3. **SLSA build provenance** — a [SLSA v1][slsa] attestation, generated by
   [`actions/attest-build-provenance`], that ties each artifact back to the
   exact workflow run, commit SHA, and source repository that produced it.

The layers are complementary: the checksum proves the bytes match, the
cosign signature proves those bytes were signed by the AegisPQ release
workflow, and the provenance attestation proves what built them and from
which commit.

## What you need

- [`cosign`] v2.2 or newer — `brew install cosign`, `go install github.com/sigstore/cosign/v2/cmd/cosign@latest`, or see the [release page](https://github.com/sigstore/cosign/releases).
- [GitHub CLI] `gh` v2.49 or newer — for `gh attestation verify`.
- Standard Unix tools (`sha256sum`, `tar`) or their Windows/PowerShell equivalents.

## 1. Verify checksums

Download the archive for your platform plus the `SHA256SUMS` file from the
release's "Assets" section, then:

```bash
sha256sum --check --ignore-missing SHA256SUMS
```

Expected output: one `OK` line per archive you downloaded. If you get
`FAILED`, stop — the file has been corrupted or tampered with.

## 2. Verify the cosign signature of SHA256SUMS

Verifying `SHA256SUMS` itself is enough to transitively trust every archive,
because step 1 binds each archive to its digest in that file.

Download `SHA256SUMS`, `SHA256SUMS.sig`, and `SHA256SUMS.pem` from the
release, then run:

```bash
cosign verify-blob \
  --certificate       SHA256SUMS.pem \
  --signature         SHA256SUMS.sig \
  --certificate-identity-regexp \
      '^https://github\.com/klanssrklbp/AegisPQ/\.github/workflows/release\.yml@refs/tags/v' \
  --certificate-oidc-issuer  https://token.actions.githubusercontent.com \
  SHA256SUMS
```

Expected output: `Verified OK`.

What you are asserting with those flags:

- `--certificate-identity-regexp` — the certificate's subject SAN must
  identify the `release.yml` workflow in
  `klanssrklbp/AegisPQ`, triggered by a version tag (`refs/tags/v…`).
  This is the most important flag: without it, cosign accepts a signature
  from *any* valid Sigstore certificate.
- `--certificate-oidc-issuer` — the certificate must have been issued in
  response to a GitHub Actions OIDC token, not some other identity provider.

You can also verify each archive directly (e.g. only `SHA256SUMS.sig` is
missing) by passing its own `.sig` / `.pem` and the archive as the last
argument.

## 3. Verify SLSA build provenance

The provenance attestation is stored in GitHub's attestations API, not as
a release asset. Verify it with the GitHub CLI:

```bash
gh attestation verify aegispq-v0.1.0-x86_64-unknown-linux-gnu.tar.gz \
  --repo klanssrklbp/AegisPQ
```

Expected output: `Loaded digest … Verification succeeded!` plus a summary
of the workflow run that produced the artifact. `gh attestation verify`
checks:

- The artifact digest matches what was attested.
- The attestation was signed by Sigstore.
- The builder is GitHub-hosted runners.
- The source repo is `klanssrklbp/AegisPQ`.

For air-gapped verification, download the attestation bundle first with
`gh attestation download <file> --repo klanssrklbp/AegisPQ`, then pass
`--bundle <file>.sigstore` to `gh attestation verify` on the offline host.

## Reporting a failed verification

If any of the three steps above fails on a release you downloaded directly
from the [GitHub releases page](https://github.com/klanssrklbp/AegisPQ/releases),
**do not run the binary** and report it per [`SECURITY.md`](../SECURITY.md).
